---
name: Python sidecar deployment pattern
description: How the FastAPI (uvicorn) service runs alongside Node.js in Replit vs Docker/ECS
---

## The Problem
Replit's production deployment manager (seccomp port-detection) cannot see ports opened by a subprocess spawned by `bash -c`. The old `--workers 2` pattern (uvicorn parent process → worker children) made the bind() call invisible to the artifact manager, causing a crash loop.

## The Solution
Run Python API as a Node.js **child_process sidecar** so only ONE port (8080) needs to be registered with the artifact manager.

## Gate Condition (critical)
In `artifacts/api-server/src/index.ts`:
```typescript
if (process.env["PYTHON_SIDECAR_ENABLED"] === "true" || process.env["NODE_ENV"] !== "production") {
```
- **Dev** (NODE_ENV=development): sidecar auto-starts ✅
- **Replit production** (NODE_ENV=production, PYTHON_SIDECAR_ENABLED=true): sidecar starts ✅
- **Docker/ECS** (NODE_ENV=production, env var absent): sidecar skipped ✅ — Python runs as separate ECS container

## Environment Variables (set in artifact.toml production run env)
- `PYTHON_SIDECAR_ENABLED=true`
- `PYTHON_BIN=/home/runner/workspace/.pythonlibs/bin/python3`
- `PY_API_DIR=/home/runner/workspace/shri-academy-api`

**Why:** Node child_process does NOT inherit the Nix shell PATH. Must use absolute path for python3.

## Shutdown Logic (important)
- Use `isShuttingDown` flag to prevent restart loop during graceful shutdown
- Handle both `error` (spawn failure) and `exit` events for restart
- Register SIGTERM/SIGINT handlers ONCE outside restart loop (avoid MaxListeners)
- Close HTTP server before exiting

## Dockerfile Issues (pnpm v10 breaking changes)
- `corepack prepare pnpm@9` → must use `pnpm@10` to match lockfile generated by pnpm v10
- `pnpm deploy --prod` → must add `--legacy` flag in pnpm v10+

**Why:** `ERR_PNPM_LOCKFILE_CONFIG_MISMATCH` and `ERR_PNPM_DEPLOY_NONINJECTED_WORKSPACE` are both pnpm v9→v10 breaking changes that only surface in Docker builds (CI uses frozen-lockfile).
